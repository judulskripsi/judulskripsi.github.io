10. Jelaskan perbedaan konsep formal dan informal dalam mendesain web


Perbedaan konsep formal dan informal dalam mendesain web adalah dari sisi tampilan, pemilihan huruf, hingga pemilihan warna pada website kita. Seperti kita kita ketahui, website pada era sekarang sudah bisa mewaikili suatu perushaan atau seseorang dalam dunia maya. Beberapa pemerintahan daerah hingga pusat juga sudah memiliki website mereka masing-masing sebagai perwakilan mereka di dunia internet.

Pembahasan

Website sendiri memiliki arti sebagai sebuah halaman pada internet yang berisikan informasi bisa tulisan, gambar hingga file multimedia lainnya. Dengan memiliki website kita bisa menginformasikan mengenai perusahaan kita dengan mudah. Dalam pembuatan website terdapat dua jenis konsep yaitu formal dan informal. Beberapa hal yang perlu diketahui diantara keduanya adalah sebagai berikut

Formal biasanya memiliki layout website yang kaku, tampilannya simetris, sedangkan pada informal tampilan website tidak kaku bahkan cenderung unik. Tata letak menu juga masuk ke dalam layout website.Pada konsep formal, penggunaan huruf biasanya menggunakan huruf yang sering digunakan seperti serif. Pada konsep informal pemilihan huruf cenderung menggunakan huruf baru.Pemilihan warna pada monsep formal menggunakan warna yang formal, pada konep informal pemilihan warna lebih berani.

Pemilihan konsep formal maupun informal dalam mendesain website selain bertujuan untuk mempercantik tampilan website juga memiliki tujuan agar website tersebut mudah untuk dikunjungi dan pencarian informasi pada wesbite tersebut mudah dilakukan.

Pelajari lebih lanjut

Demikian pembahasan mengenai perbedaan konsep formal dan informal dalam mendesain website, perbedaan mendasar terdapat pada sisi layout, pemilihan huruf dan warna. Untuk mempelajari materi lebih lanjut mengenai pembuatan website dapat dibaca pada link berikut

12. tuliskan dan jelaskan konsep kerja halaman web dapat di tampilkan dalam web browser komputer klien​


Jawaban:

Cara Kerjanya Web Server adalah

Sederhananya tugas web server adalah untuk menerima permintaan dari client dan mengirimkan kembali berkas yang diminta oleh client tersebut.

Seperti yang sudah saya informasikan diatas bahwa client yang dimaksud disini adalah komputer desktop yang memiliki atau telah menginstall web browser seperti Chrome, Mozilla, Opera dan lain-lain yang dapat terhubung ke web server melalui jaringan internet atau intranet.

Perangkat lunak web server terdapat pada komputer server, dan di komputer ini pula-lah data-data website tersimpan dengan rapih. Sama halnya dengan komputer client, komputer server juga harus terhubung dengan jaringan internet atau jaringan intranet untuk dapat diakses oleh client.

Pada saat client (browser) meminta data web page kepada server, maka instruksi permintaan data oleh browser tersebut akan dikemas di dalam TCP yang merupakan protokol transport dan dikirim ke alamat yang dalam hal ini merupakan protokol berikutnya yaitu Hyper Text Transfer Protocol (HTTP) dan atau Hyper Text Transfer Protocol Secure (HTTPS). Data yang diminta dari browser ke web server disebut dengan HTTP request yang kemudian akan dicarikan oleh web server di dalam komputer server. Jika ditemukan, data tersebut akan dikemas oleh web server dalam TCP dan dikirim kembali ke browser untuk ditampilkan. Data yang dikirim dari server ke browser dikenal dengan HTTP response. Jika data yang diminta oleh browser tersebut ternyata tidak ditemukan oleh web server, maka web server akan menolak permintaan tersebut dan browser akan menampilkan notifikasi error 404 atau Page Not Found.

Meskipun proses atau cara kerja web server diatas seperti sangat rumit, tapi pada prakteknya proses tersebut berlangsung dengan sangat cepat. Anda bahkan bisa sampai tidak menyadari bahwa pada saat anda meminta suatu halaman web, ternyata hal itu membutuhkan proses yang sangat panjang sampai halaman tersebut dapat anda lihat di browser anda.

14. 3. Jelaskan tentang konsep web-caching 4. Apakah web caching dapat mengurangi dealay untuk semua objek yang di riques oleh user? Mengapa?


no 4. Manfaat utama dari penggunaan cache adalah mengurangi beban kinerja server, dengan adanya cache webserver/database server tidak perlu melakukan proses yang sama untuk permintaan yang sama.Web server akan menggunakan data dari arsip cache yang selanjutnya diberikan kepada client sebagai respon. Hal ini sangat berguna jika website kita mempunyai trafik yang tinggi setiap harinya, penggunaan memory pada server bisa kita gunakan untuk proses yang lain. Selain itu respon terhadap request client menjadi lebih cepat.Semua web cache memiliki satu set aturan yang mereka gunakan untuk menentukan kapan tidaknya melayani obyek dari cache. Beberapa aturan ini diatur di protokol (HTTP 1.0 dan 1.1), dan ada pula yang ditetapkan oleh administrator cache (baikpengguna cache browser, atau administrator proxy).
